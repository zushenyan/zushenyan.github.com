How to Write Testable AJAX Actions

2016-12-12
#f2e #javascript #functional_programming

Let’s say, you are working on an AJAX call for your project:

And your action probably looks like this:

Here comes a question, how do you test your fetchUserAction?

Most answers will be mock/stub the AJAX call, and there are many tools for mock/stub:

  • jasmine-ajax
  • sinon
  • nock
  • Or if you are using Jest you have to mock js files first.
  • Blah blah blah… countless frameworks, you name it.

These frameworks, the mock/stub technique, to sum up, is just an approach to intercept function call or replace the original object for a fake response.

And now, if you are a lazy person like me, you might be wondering “why the fuck I am doing this?”. I mean, look at how many steps and documents you have to go through for testing async actions? Is this really necessary?

Bad news, yes, we still have to finish the test (because we are a good programmer). However this time we will take a much more painless, flexible approach.

In functional programming, the AJAX call has side effect. It makes functions impure if it lives in them. Impure makes functions uncertain because we can’t get static results even if we feed the same input every time.

For making our action testable, we have to isolate the side effect.
